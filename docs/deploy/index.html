Tus variables rellenar una vez

Los bloques de código de abajo usan estos valores. Cambia cualquiera y se actualizan en vivo.

1
Crear cuenta en Google Cloud
  1. Entra a console.cloud.google.com con tu cuenta Google.
  2. Acepta los términos del servicio.
  3. Activa la capa gratuita / créditos de estudiante. Si usas la cuenta institucional, verifica que tengas permisos de facturación.
  4. Crea o vincula una cuenta de facturación (requisito incluso para el free tier). Menú ☰ → Billing.
Ojo: Cloud Run y Artifact Registry requieren facturación activa. El free tier cubre el uso típico del curso, pero apaga los servicios al terminar.
2
Crear proyecto y elegir PROJECT_ID
  1. Arriba a la izquierda, clic en el selector de proyecto → New project.
  2. Nombre sugerido: hello-world-quarkus. El PROJECT_ID debe ser único globalmente — anótalo.
  3. Vincula la cuenta de facturación al proyecto.

Copia tu PROJECT_ID al panel superior. Todos los siguientes comandos lo usan.

3
Habilitar APIs requeridas

Opción A — UI: Menú ☰ → APIs & Services → Enable APIs. Busca y habilita:

  • Artifact Registry API
  • Cloud Build API
  • Cloud Run Admin API
  • Secret Manager API
  • Cloud SQL Admin API (si usarás Cloud SQL)

Crear Trigger en Cloud Build (auto-build en cada push)

  

    
Si conectaste GitHub, crea un trigger para construir automáticamente:

    2. 
      
  2. Source: conecta tu repo de GitHub (autoriza la app de Google Cloud Build).
    3. 
      
  3. Event: Push to a branch. Branch regex: ^main$.
    4. 
      
  4. Configuration: Cloud Build configuration file (yaml) → cloudbuild.yaml.
    5. 
      
  5. Substitution variables (si quieres sobrescribir los defaults):
        
      
          
    • _REGION = us-central1
      • 
          
    • _REPOSITORY = hello-world-repo
      • 
          
    • _IMAGE = hello-world
      • 
        
    
      
    6.

Desplegar a Cloud Run con variables de entorno

  

    
UI: Menú ☰ → Cloud Run → Deploy container → Service.

    
    
      
  1. Container image URL: selecciona desde Artifact Registry
        
    2. 
      
  2. Service name: hello-world
    3. 
      
  3. Region: us-central1
    4. 
      
  4. Authentication: Allow unauthenticated invocations (para pruebas).
    5. 
      
  5. Container port: 8080
    6. 
      
  6. Pestaña Variables & Secrets → agrega:
        
      
          
    • DB_KIND = mysql
      • 
          
    • DB_USERNAME = (tu usuario)
      • 
          
    • DB_PASSWORD = (desde Secret Manager — Reference a secret)
      • 
          
    • DB_JDBC_URL = jdbc:mysql://...
      • 
          
    • DB_SCHEMA_STRATEGY = update
      • 
          
    • FIREBASE_SERVICE_ACCOUNT_LOCATION = /secrets/firebase.json
      • 
        
    
      
    7. 
      
  7. Pestaña Variables & Secrets → Mount as volume:
        
      
          
    • Secret: firebase-service-account
      • 
          
    • Mount path: /secrets
      • 
          
    • Filename: firebase.json
      • 
        
    
      
    8. 
      
  8. Si usas Cloud SQL: pestaña Connections → añadir la instancia.
